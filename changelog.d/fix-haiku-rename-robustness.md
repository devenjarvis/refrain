### Fixed

- Haiku-driven branch names and task summaries now succeed far more often. The `claude` subprocess invoked for each Haiku call now runs with a slim argv that disables MCP discovery, slash-command resolution, session persistence, built-in tools, and dynamic system prompt sections — and adds `--bare` when `ANTHROPIC_API_KEY` is set — which dramatically cuts cold-start time. Per-attempt and overall retry budgets were widened to absorb the residual long tail (45s/140s for branches, 45s/120s for summaries), and slug truncation now stops at the last `-` boundary inside the first 40 chars rather than mid-word. The branch namer's retry path now drives off the session's original prompt rather than whatever follow-up prompt happened to land next, and the task summarizer was rewired through the same retry helper so failures get logged to `.baton/logs/haiku.log` (now tagged `kind=branch` / `kind=summary`) instead of being silently swallowed.
