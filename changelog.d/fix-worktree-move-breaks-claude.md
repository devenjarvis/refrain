### Fixed

- The first-prompt branch rename no longer relocates the on-disk worktree. The previous release ran `git worktree move` so `.refrain/worktrees/<random>` would re-align with the Haiku-generated slug, but moving the directory out from under the running Claude process broke its session: the absolute `--settings` path baked into Claude's argv stopped resolving, the process's `PWD` env went stale (so subprocesses Claude spawned for Bash tool calls inherited a non-existent cwd), and Claude's session files — indexed by the original working directory — silently diverged. Sessions become unable to continue. The branch rename is still applied (HEAD symref updates atomically without disturbing the worktree, as before), and the dashboard's session name now reflects the new branch, but the worktree path stays at its original adjective-noun until the session is closed.
