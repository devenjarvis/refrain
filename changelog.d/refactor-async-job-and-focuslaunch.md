### Changed

- Consolidated the four hand-rolled async-task state machines on `agent.Session` (rename, task-summary, plan-draft, plan-revise) onto a single internal `asyncJob` field-bundle in `internal/agent/asyncjob.go`. Each session now stores one `asyncJob` per task (running flag + optional cancel func + last error + optional retry-attempt progress) instead of five scattered fields per task. Public methods (`IsDrafting`, `TryStartDraft`, `CancelDraft`, `DraftError`, etc.) keep their existing signatures; only the storage shape changed. Cross-job exclusivity (revising blocks while drafting is in flight) stays atomic — every wrapper still acquires `Session.mu` once and reads/writes through the asyncJob fields under the lock. Adding a new async task is now one `asyncJob` field plus thin wrappers, instead of five copy-pasted state fields and a half-dozen near-identical helpers.

- Pulled the focusLaunch terminal overlay's keypress handler out of `App.updateDashboard` into `App.updateFocusLaunchKeys` (and `App.closeFocusLaunchAgent` for the ctrl+w path). `updateDashboard` is ~160 lines shorter and now reads as a router between overlay handlers rather than inlining one of them. No behavior change.
