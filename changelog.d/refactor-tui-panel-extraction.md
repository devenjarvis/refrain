### Changed

- TUI internals: extracted per-panel state and dispatch from `internal/tui/app.go`'s monolithic ~3950-line `Update` function. A new `PanelModel` interface (with a `PanelServices` seam carrying lookups, navigation callbacks, and Cmd factories) lets panels live in their own files with isolated state, while App's outer Update keeps cross-cutting message routing.
  - `reviewPanelModel` now owns the review-panel state (task cursor, diff viewport, spec overlay, per-task diff cache) and its 210-line key/click dispatch; only the lifetime-spanning `reviewDiffCache` stays on App.
  - `shippingPanelModel` owns the shipping-panel state (feedback cursor, detail scroll) and its 110-line key dispatch plus the nested feedback-note modal; `feedbackTriage` (which survives panel close/reopen) stays on App.
  - `FocusedCursor` replaces the four parallel per-section index fields (`focusPlanningIdx` / `focusBuildingIdx` / `focusReviewIdx` / `focusShippingIdx`) and `focusCursorSection` on App plus their mirror fields on `dashboardModel`, with six panic-guarded helper functions (`focusSectionIdx`, `focusSectionItems`, `focusSectionCounts`, `clampFocusCursor`, `moveFocusCursorUp`, `moveFocusCursorDown`) collapsing into methods on the cursor type. Dashboard exposes `sectionItems` / `sectionCounts` so cursor logic and renderers share one source of truth.
  - The panel-close ritual is centralised via `PanelServices.ClosePanel` (a synchronous callback closing over App's local copy in the current Update tick), replacing ~30 scattered `panelFocus = focusList; reviewSession = nil` pairs. App snapshots the active panel before delegating and only re-assigns if `ClosePanel` didn't fire, so close and update can't race on the same tick.
  - The `addressReviewFeedback` and `addressFeedback` flows become messages (`reviewReworkRequestMsg`, `shippingFeedbackRequestMsg`) emitted by the panels and handled by App, keeping cache mutation and agent spawning on the cross-cutting layer where they belong.
  - `app.go` drops from 5700 to 5231 lines (-469); 832 lines move into focused panel files (`focuscursor.go`, `panel.go`, `reviewpanelmodel.go`, `shippingpanelmodel.go`). Eight panel-level unit tests demonstrate the new test pattern — instantiate the panel directly with a stub `PanelServices`, drive it with messages, assert state — no full App spin-up, no goroutines, no claude binary. All TUI tests pass with the race detector.

- No user-facing behaviour changes: panel keys, navigation, mouse handling, render output, and cross-panel transitions (auto-promote review→shipping on PR creation, PR-closed cleanup, merge → Complete) are preserved.
